# 📉 Customer Churn Predictor

A machine learning web app that predicts whether a telecom customer is likely to churn, comparing five classification algorithms and surfacing the key business drivers behind churn.

## 📋 Problem Statement

Customer churn directly impacts recurring revenue — acquiring a new customer typically costs far more than retaining an existing one. This project builds a churn-risk assessment tool that predicts whether a customer will leave, so a business can proactively target them with retention offers before they cancel.

## 📊 Dataset

- **Source:** IBM Telco Customer Churn dataset (`telco_churn.csv`)
- **Size:** 7,043 customers, 19 features + 1 target (`Churn`)
- **Features:** Demographics (gender, senior citizen, partner, dependents), account info (tenure, contract type, payment method, paperless billing), services subscribed (phone, internet, online security, tech support, streaming), and billing (monthly/total charges)

## 🛠️ Tech Stack

- **Language:** Python
- **ML:** scikit-learn (Logistic Regression, Random Forest, Gradient Boosting, KNN, Decision Tree)
- **UI:** Streamlit
- **Visualization:** Plotly
- **Data handling:** Pandas, NumPy

## ⚙️ Methodology

1. **Data cleaning** — dropped `customerID` (identifier, not predictive); `TotalCharges` was stored as text with 11 blank rows (new customers with 0 tenure), coerced to numeric and filled with 0
2. **Encoding** — all categorical features (gender, contract type, internet service, payment method, service add-ons, etc.) one-hot encoded
3. **Scaling** — all features standardized using `StandardScaler`
4. **Train/test split** — 80/20 stratified split
5. **Model training** — five classifiers trained and evaluated: Logistic Regression, Random Forest, Gradient Boosting, K-Nearest Neighbors, Decision Tree

## 📈 Results

| Model | Accuracy | ROC AUC |
|---|---|---|
| Logistic Regression | 80.7% | 84.2% |
| Gradient Boosting | 79.8% | 84.2% |
| Random Forest | 79.3% | 82.5% |
| K-Nearest Neighbors | 74.7% | 77.2% |
| Decision Tree | 73.5% | 65.6% |

**Best performing model:** Logistic Regression (highest accuracy and tied for best ROC AUC)

## 🔍 Key Insight: Why Models Disagree on the Same Customer

Just like the heart disease model, running the *same* customer profile through all five algorithms can produce different verdicts and confidence levels — this is expected, not a bug.

Each algorithm learns a fundamentally different strategy from the same data:
- **Logistic Regression** draws a smooth, roughly linear decision boundary across all features
- **Decision Tree** asks a sequence of sharp yes/no questions (e.g., "Is Contract = Month-to-month? → Is tenure < 12?"), which is why it often gives very confident (0%/100%) answers
- **KNN** has no formula — it just looks at the nearest customers in the training data and votes
- **Random Forest / Gradient Boosting** average or chain together many small decision trees, smoothing out sharp edges

On clear-cut customers (e.g., long-tenure, two-year contract, low charges → obviously loyal), all models tend to agree. Disagreement surfaces mainly on **borderline customers** sitting close to each model's decision boundary — which is itself useful signal, since it flags accounts worth a closer manual look rather than trusting one automated score blindly. The Model Comparison tab (ranked by ROC AUC) tells you which model to weight most heavily when verdicts conflict.
